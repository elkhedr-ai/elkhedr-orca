# Orca Reports

In a standalone Orca clone, major delivery reports belong in this app-local folder first.
When the parent workspace is available, mirror them to `../../../docs/reports/`.

Use report names like:

```text
YYYY-MM-DD_ELK-ORCA-PROD-###_DELIVERY_REPORT.md
```

Each report should cite completed roadmap rows, action approval impact, MCP/API impact,
verification commands, security controls, and known risks.
